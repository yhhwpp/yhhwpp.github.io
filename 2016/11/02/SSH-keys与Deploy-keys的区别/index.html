SSH keys与Deploy keys的区别

发表于 2016-11-02 | 分类于 Github | 阅读次数:

在 Github 上建了多个仓库，本人之前的一贯做法是，把ssh目录下的id_rsa.pub公钥copy至repository的Deploy keys，这样会导致另外一个repository无法增加Deploy keys，会提示Key is already use的错误。正确的做法是将id_rsa.pub添加至账户的SSH keys。

SSH keys 与 Deploy keys 的区别：

  • SSH keys 是账户的最高的key，只要有账户的权限，其下的repository也是有权限访问的
  • repository 的 Deploy keys， 是repository 的专有key
  • 通俗的区别就是SSH keys 相当于 一个大房子的钥匙（这个钥匙可以打开屋里任意房间），而Deploy keys相当于房子里一个房间的钥匙

另外id_rsa.pub是公钥，可以添加至服务器段。本机的id_rsa是与之对应的私钥，密钥是成对的。
